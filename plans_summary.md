# Nexus 项目演进计划汇总

## 阶段 1：基础架构确立
- **目标**: 实现基础的 SSH 指令下发与结果展示。
- **方案**: Flask + Paramiko (RESTful 模式) + xterm.js 基础渲染。
- **状态**: 已完成 (由方案 2 替代升级)。

## 阶段 2：实时全功能终端重构
- **目标**: 提供类似 Xshell 的流式交互体验。
- **方案**: 引入 `Flask-SocketIO` + `PTY` 伪终端，实现 WebSocket 双向实时透传。
- **状态**: 已完成。

## 3. 资产管理与 CRUD 系统加固
- **目标**: 解决资产添加、编辑、删除失效及动态渲染下的点击丢失问题。
- **方案**: 引入“全局事件代理”架构，增加 `group` 分组模型，补全前端语义容器。
- **状态**: 已完成。

## 4. 集成式 SFTP 与云端编辑器
- **目标**: 实现文件可视化管理与在线高亮编辑。
- **方案**: 后端封装 SFTP 原子 API，前端集成 Ace Editor，支持 Python/YAML/Shell 多模式高亮。
- **状态**: 已完成。

## 5. 多会话并行（Multi-Tab）与弹性布局
- **目标**: 支持多服务器同时在线，且布局可自由调节。
- **方案**: 构建 `SessionManager` 状态栈，实现 Tab 隔离；引入 12px 热区 Resizer 拖拽引擎。
- **状态**: 已完成。

## 6. 连接稳定性与兼容性加固
- **目标**: 解决真机 SFTP 挂起与 Mock 环境演示问题。
- **方案**: 注入 `eventlet` 猴子补丁，增加 `keepalive` 脉冲，为 `127.0.0.1` 提供演示 Mock。
- **状态**: 已完成。

## 7. 模块化 UI 增强与性能监控中心
- **目标**: 实现“监控”、“脚本库”、“知识中心”的模块化渲染，提供实时性能看板。
- **方案**: 前端采用“模块视图缓存+动态渲染”架构；后端集成 `psutil` 采集实时指标。
- **状态**: 进行中。

## 8. 布局弹性与稳定性修复 (Resizer Fix)
- **目标**: 解决 FTP 分栏拖动时的布局偏移抖动问题，提升操作手感。
- **方案**: 优化 `onMouseMove` 计算逻辑，增加容器锁定机制，防止 Grid 布局级联抖动。
- **状态**: 已完成。

## 9. 全局 IDE 级弹性布局 (Multi-Resizer)
- **目标**: 实现左侧资源树与右侧 AI 面板的宽度自由调节。
- **方案**: 引入 `v-resizer` 垂直拖拽热区，通过 JS 动态调整 Grid 轨道比例。
- **状态**: 已完成。

## 10. 布局收口与状态栏补全
- **目标**: 修复底部黑边，提供全局状态展示（如连接节点名、编码、版本）。
- **方案**: 补全 `status-bar` 结构组件，优化 Grid `grid-template-rows` 填充稳定性。
- **状态**: 已完成。

## 11. 渲染性能与隔离优化 (Final Fix)
- **目标**: 彻底消除跨列拖拽联动，解决局部渲染留白。
- **方案**: 引入 `min-width: 0` 锁定 Grid 轨道；优化 Resize 事件防抖处理；修复 `this` 作用域导致的 JS 崩溃。
- **状态**: 已完成。
## 12. 深度集成：账户系统与 AI 模型配置
- **目标**: 实现双栏 IDE 风格的设置中心，打通 AI Copilot 的参数配置链路。
- **实施方案**:
    - **UI 架构**: 采用“左侧目录+右侧主舞台”的响应式双栏布局。利用动态路由 `renderSettingsForm` 实现内容热切。
    - **账号模块**: 默认实名 `FuQian` (Pro)，展示超级管理员权限及系统到期状态。支持后端 `/api/settings` RESTful 同步。
    - **通用模块**: 
        - **主题切换**: 支持 `dark/light` 模式。实现原理为 JS 动态操控 `body` 的 `theme-light` CSS 类，覆盖全局 CSS 变量。
        - **显示偏好**: 预留多语言接口。
    - **模型模块**: 
        - **多源支持**: 支持 OpenAI、Ollama 独立 Endpoint 配置。后端采用 `requests` 实现基于标准 OpenAI 协议的转发。
        - **安全性**: 采用 `type="password"` 渲染 API Key，防泄漏。
        - **即时保存**: 采用全量 JSON 载荷提交后端持久化。
    - **交互对接**: 补全了 `main.js` 缺失性事件监听，实现了从回车发送到流式渲染的完整闭环。
    - **主题适配**: 彻底修复了底部状态栏 `status-bar` 在亮色模式下的黑边残留。
- **状态**: 已完成。

## 13. AI 对话深度优化与模型热切 (Current)
- **目标**: 提升对话界面的视觉层次感，并支持在面板头部快速切换底层模型。
- **实施方案**:
    - **气泡重塑**: 区分 User (蓝色靠右) 与 AI (深灰靠左)，引入非对称圆角与细微阴影渲染。
    - **头像系统**: 为 AI 引入 `bot/sparkles` 渐变图标，为用户提供首字母 `F` 标识，增强交互仪式感。
    - **代码块美化**: 针对 AIOps 场景优化 Markdown 代码块，提供深色、等宽、易读的代码渲染环境。
    - **模型选择器**: 在 AI 面板 Header 集成 `model-mini-select` 下拉框，联动后端自动更新 `ai_model` 配置。
    - **输入控制**: 支持 Shift+Enter 换行，Enter 发送，输入框随字数动态垂直增长高度。
- **状态**: 已完成。

## 14. 模型分组管理与极简对话 UI (Latest)
- **目标**: 深度模拟 Trae 主流 IDE 体验，将模型配置从单一表单转变为分类管理，并优化对话交互密集度。
- **实施方案**:
    - **模型分组管理**: 设置页面重构，显示“内置”与“自定义”模型列表。引入折叠分类与表格化展示（模型名、服务商、操作）。支持“添加模型”弹窗。
    - **对话 UI 精益化**:
        - **选择器下移**: 将模型切换按钮从面板头部移至输入框右下角发送按钮旁。
        - **输入框图标化**: 整合 `@` (提及)、`#` (上下文)、`Img` (快照) 等功能图标占位符，增强视觉丰富度。
        - **微型化按钮**: 收缩发送按钮尺寸，移除冗余文字，仅保留图标。
- **状态**: 已完成。

## 15. 精致化视觉与功能闭环 (Current)
- **目标**: 优化“添加模型”与“发送对话”按钮的审美观感，并补全模型添加的全流程交互逻辑。
- **实施方案**:
    - **按钮重塑 (Polished Buttons)**: 
        - `btn-confirm-sm`: 移除厚重的填充色，改为微深背景搭配淡入淡出边框，视觉更稳重。
        - `send-btn`: 调整为圆角矩形或圆形极简图标（如 `arrow-up`），移除生硬的蓝色块感。
    - **模型添加弹窗 (Model Config Modal)**: 真正实现 `model-config-modal` 结构。支持输入 Provider, Endpoint, Key 及 Model ID。
    - **逻辑闭环**: 点击“添加模型”唤起弹窗，点击保存通过 `/api/settings` 持久化，并自动刷新前端管理列表。
- **状态**: 已完成。
